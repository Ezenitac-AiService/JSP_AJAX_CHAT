# PRD: E2E 테스트 및 도커 기반 실제 구동 검증

## 1. 프로젝트 개요
- **목적**: 리팩토링된 AJAX 채팅 애플리케이션의 실제 동작 여부를 도커 환경에서 완벽하게 검증(E2E)하고, 훈련생들에게 모던한 테스트 자동화 과정을 제시함.
- **주요 목표**: 
  - 전체 애플리케이션(Application + DB)을 도커 컨테이너화.
  - Playwright를 사용한 실시간 다중 사용자 채팅 기능 검증.
  - Chrome DevTools를 활용한 네트워크 규격(JSON) 최종 확인.

## 2. 기술적 요구사항
- **Containerization**: 
  - `tomcat:9-jdk17` 이미지를 사용하여 JSP 애플리케이션 서버 구성.
  - 기존 MySQL 컨테이너와 네트워크 연결.
- **E2E Testing (Playwright)**:
  - 브라우저 인스턴스 2개를 생성하여 사용자 A와 B 간의 메시지 송수신 테스트.
  - 메시지 전송 후 DOM에 말풍선이 생성되는지 CSS 셀렉터로 확인.
- **Validation**:
  - AJAX 요청 시 응답 데이터가 실제 JSON 포맷인지 검증.

## 3. 테스트 시나리오 (Scenario C)
1. **User A 접속**: `join.jsp`에서 'UserA'로 입장.
2. **User B 접속**: 새로운 브라우저 컨텍스트에서 'UserB'로 입장.
3. **User A 발신**: "Hello from A" 메시지 전송.
4. **User B 확인**: User B의 화면에 User A의 메시지가 실시간(Polling 주기 내)으로 나타나는지 확인.
5. **User B 회신**: "Hi A, I'm B" 메시지 전송.
6. **User A 확인**: User A의 화면에 User B의 메시지가 나타나는지 확인.

## 4. 기대 결과
- [ ] 모든 컨테이너가 정상적으로 구동되고 앱이 서빙됨.
- [ ] 두 사용자 간의 메시지 전달 성공.
- [ ] 테스트 자동화 리포트 생성.
